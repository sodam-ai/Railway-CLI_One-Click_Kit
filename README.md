# Railway 원클릭 키트 (Railway CLI One-Click Kit)

**Railway CLI**(내가 만든 앱·서버를 인터넷에 올려 실행해 주는 도구)를 클릭 몇 번으로 **설치 · 사용 · 제거**할 수 있게 만든, 컴퓨터가 처음인 분을 위한 키트입니다. 명령어를 외울 필요가 없습니다. 한국어 안내판이 **"지금 누를 번호"** 를 직접 알려줍니다.

> **Railway(레일웨이)란?** 내가 만든 앱·서버·데이터베이스를 **인터넷에 올려 실행**하고 인터넷 주소(URL)를 받게 해 주는 서비스입니다. (예: 챗봇, 백엔드, API 서버, 디스코드 봇)
>
> 📄 이 문서(README.md)와 **완전히 같은 내용의 PDF** 가 함께 들어 있습니다 → `README.pdf`
> 🇬🇧 English: `README.en.md` / `README.en.pdf`
> 👶 더 쉬운 그림 따라하기: `사용설명서.md` / `사용설명서.pdf`

## 1. 빠른 시작 (이것만 기억하세요)

1. 폴더 안의 **`시작하기.bat`** 를 **더블클릭**합니다.
2. 화면이 **파란 배경으로 강조해 주는 번호**를 그대로 누릅니다. (위/아래 화살표 + Enter, 또는 숫자키)
3. 보통 순서: **[1] 설치 → [2] 로그인 → [5] 자주 쓰는 작업** 에서 배포·로그.

끝입니다. 나머지는 안내판이 알아서 도와줍니다. 막히면 **[9] 내 상태 점검** 을 누르세요.

## 2. 사전 준비물 · 필요 프로그램

| 항목 | 필수? | 설명 |
|---|---|---|
| 윈도우 10 또는 11 | ✅ 필수 | 이 키트는 윈도우 전용입니다. |
| 인터넷 연결 | ✅ 필수 | 설치·로그인·배포에 필요합니다. |
| Railway 계정 | ✅ 필수(무료 가입) | https://railway.com 에서 무료 가입. 로그인할 때 브라우저로 연결됩니다. |
| **Node.js** (LTS) | ⛅ 권장(없어도 됨) | 있으면 가장 매끄럽게 설치됩니다. **없어도** 키트가 자동으로 다른 방법(프로그램 파일 직접 내려받기)을 시도합니다. https://nodejs.org 의 **LTS** 버전 권장. |
| 관리자 권한 | ❌ 불필요 | **이 키트는 관리자 권한이 필요 없습니다. 윈도우 "관리자 허용(UAC)" 창은 뜨지 않습니다.** (프로그램이 내 계정 폴더에만 설치되기 때문입니다.) |

## 3. 다운로드 방법 · 차단 해제

1. 받은 **zip(압축) 파일**을 원하는 폴더에 둡니다.
2. 압축을 풀기 **전에**, zip 파일을 **우클릭 → 속성 → 맨 아래 '차단 해제(Unblock)' 체크 → 확인**. (인터넷에서 받은 파일을 윈도우가 잠가두는 것을 푸는 단계입니다.)
3. 압축을 풉니다(우클릭 → 압축 풀기).
4. 폴더 안의 **`시작하기.bat`** 를 더블클릭합니다.
   - 파란 **"Windows의 PC 보호"** 창이 뜨면 → **추가 정보 → 실행**.
   - `시작하기.bat` 가 한 번 실행되면, 폴더 안 나머지 파일의 잠금은 **자동으로 풀립니다.**

## 5. 홈 메뉴 한눈에 보기 (`시작하기.bat`)

`시작하기.bat` 를 실행하면 아래 메뉴가 뜹니다. 위에는 **내 컴퓨터 상태**(Node.js / Railway / 로그인)와 **"지금 할 일"** 한 줄이 함께 표시됩니다.

| 번호 | 메뉴 | 설명 |
|---|---|---|
| **1** | 설치 / 업데이트 | Railway CLI를 깔거나 최신으로 올립니다. |
| **2** | 로그인 | 브라우저로 내 Railway 계정에 연결합니다. |
| **3** | 로그인 상태 확인 | 지금 누구로 로그인됐는지 보여줍니다. |
| **4** | 내 프로젝트 보기 | 내 Railway 프로젝트 목록을 봅니다. |
| **5** | **자주 쓰는 작업** | 배포·로그·상태 등 **쉬운 메뉴**(아래 6장). |
| **6** | 제거하기 | 깨끗이 지웁니다. (내 코드는 안 지움) |
| **7** | 사용설명서 | 왕초보 가이드(`사용설명서.md`)를 엽니다. |
| **8** | Railway 대시보드 | 웹브라우저로 Railway 관리 페이지를 엽니다. |
| **9** | **내 상태 점검 / 문제 해결** | 막혔을 때 자가진단(아래 7장). |
| **0** | 끝내기 | 안내판을 닫습니다. |

## 6. [5] 자주 쓰는 작업 (왕초보용 쉬운 메뉴)

홈에서 **[5]** 를 누르면, 무서운 30개짜리 영어 메뉴 대신 **가장 많이 쓰는 작업만 모은 한국어 메뉴**가 화살표 선택으로 열립니다.

| 번호 | 메뉴 | 하는 일 | 안전장치 |
|---|---|---|---|
| 1 | 프로젝트 시작 | 이 폴더로 새 Railway 프로젝트를 만듭니다 (`railway init`) | — |
| 2 | 프로젝트 연결 | 이미 있는 프로젝트에 이 폴더를 연결 (`railway link`) | — |
| 3 | **배포하기** | 이 폴더를 **인터넷에 공개**합니다 (`railway up`) | **YES 입력 확인** |
| 4 | 다시 배포 | 최근 내용을 다시 공개 (`railway redeploy`) | **YES 입력 확인** |
| 5 | 실시간 로그 보기 | 서버 실행 기록을 실시간으로 봅니다 (`railway logs`) | 멈춤: Ctrl+C |
| 6 | 프로젝트 상태 | 지금 연결/배포 상태 (`railway status`) | — |
| 7 | 환경변수 보기 | 설정값 목록 (`railway variables`) | — |
| 8 | 전체 30기능 열기 | 고급 메뉴(`RUN.bat`)를 새 창으로 엽니다 | — |
| 0 | 뒤로 | 홈으로 돌아갑니다 | — |

> **배포(3번)는 되돌리기 어려운 작업**(인터넷 공개)이라, 실행 직전에 **대문자 `YES`** 를 한 번 더 묻습니다. 그냥 Enter를 누르면 취소됩니다.

## 7. [9] 내 상태 점검 / 문제 해결 (막혔을 때 여기!)

홈에서 **[9]** 를 누르면, 한 화면에서 아래를 **자동 점검**하고 각 항목마다 **무엇을 누르면 되는지** 한국어로 알려줍니다.

- Node.js(필수 부품) 있는지
- Railway CLI 있는지 / 버전 / 설치 위치
- 로그인 됐는지
- **인터넷 연결**(railway.com 까지 최대 4초 확인)

그리고 아래 **빠른 해결** 버튼이 있습니다.

| 번호 | 기능 | 언제 쓰나요 |
|---|---|---|
| 1 | 설치 / 업데이트 다시 하기 | 설치가 안 됐거나 망가졌을 때 |
| 2 | **PATH 새로고침** | **방금 설치했는데 "없음"으로 뜰 때** (창을 다시 안 열어도 인식되게 함) |
| 3 | 다시 점검 | 조치 후 상태를 다시 확인 |
| 0 | 뒤로 | 홈으로 |

## 8. 전체 30가지 기능 (`RUN.bat`)

고급 사용자를 위한 전체 메뉴입니다. 홈 **[5] → [8] 전체 30기능 열기** 로 열거나, `RUN.bat` 를 직접 더블클릭합니다. 각 줄은 **번호 · 한국어 · 영어** 가 함께 적혀 있습니다.

**계정(ACCOUNT)**
- 1 로그인(브라우저) · 2 로그인(브라우저 없이) · 3 로그아웃 · 4 내 계정 보기

**버전(VERSION)**
- 5 버전 보기 · 6 업데이트

**프로젝트(PROJECT)**
- 7 새 프로젝트 시작 · 8 프로젝트에 연결 · 9 연결 해제 · 10 내 프로젝트 목록 · 11 프로젝트 상태 · 12 브라우저에서 열기

**배포(DEPLOY)**
- 13 **배포(올리기)** *YES 확인* · 14 **다시 배포** *YES 확인* · 15 **서비스 내리기** *YES 확인* · 16 서비스 선택 · 17 DB/플러그인 추가

**로그 / 쉘(LOGS + SHELL)**
- 18 실시간 로그 · 19 빌드 로그 · 20 쉘 열기 · 21 명령 실행 · 22 SSH 접속 · 23 플러그인 접속

**설정(CONFIG)**
- 24 환경변수 · 25 환경 선택 · 26 커스텀 도메인 · 27 볼륨 관리

**도움말(HELP)**
- 28 문서 열기 · 29 도움말 · 30 직접 명령 입력(고급)

- 0 종료 (q / exit / 종료 입력도 가능)

> 번호에 공백이 섞여도(예: ` 13 `) 알아서 인식합니다. 없는 번호를 눌러도 그냥 메뉴로 돌아오니 걱정하지 마세요.

## 9. 표준 워크플로우 (처음부터 끝까지 흐름)

```
[다운로드 + 차단 해제]
        ↓
  시작하기.bat 더블클릭
        ↓
  [1] 설치 / 업데이트          ← 처음 한 번만
        ↓
  [2] 로그인 (브라우저)        ← 처음 한 번만
        ↓
  [5] 자주 쓰는 작업
        ├─ [1] 프로젝트 시작  또는  [2] 프로젝트 연결
        ├─ [3] 배포하기 (YES)        ← 인터넷에 공개
        ├─ [5] 실시간 로그           ← 잘 떴는지 확인
        └─ [6] 프로젝트 상태
        ↓
  (막히면) [9] 내 상태 점검 / 문제 해결
        ↓
  (다 쓰면) [6] 제거하기  ← 선택
```

## 10. 방법별 정리 (설치 · 실행 · 사용 · 제거)

### 설치 방법
1. `시작하기.bat` → **[1] 설치 / 업데이트**.
2. 검은 창(영어)이 자동으로 진행됩니다. **닫지 말고 기다리세요**(보통 1~10분).
3. `INSTALLATION COMPLETE` 또는 비슷한 완료 문구가 보이면 성공입니다.
4. Node.js가 없어도 키트가 다른 방법을 자동 시도합니다. 그래도 안 되면 https://nodejs.org 에서 LTS 설치 후 다시 [1].

### 실행 방법
- 항상 **`시작하기.bat`** 더블클릭으로 시작합니다. (한국어 안내판)
- 한국어 안내판이 안 열리면 `INSTALL.bat → RUN.bat → UNINSTALL.bat` 을 직접 더블클릭해도 동일하게 동작합니다(영어).

### 사용 방법 (배포까지)
1. **[2] 로그인** (처음 한 번).
2. 내 앱(코드) 폴더에서 **[5] → [1] 프로젝트 시작**(또는 [2] 연결).
3. **[5] → [3] 배포하기** → 대문자 `YES` 입력 → 인터넷 공개.
4. **[5] → [5] 실시간 로그** 로 잘 돌아가는지 확인.

### 제거 방법
1. `시작하기.bat` → **[6] 제거하기**.
2. 무엇이 지워지는지 확인 후 **대문자 `DELETE`** 입력.
3. 프로그램만 깨끗이 지워집니다. **내 코드 파일·Node.js·Railway 서버의 내 앱은 그대로** 둡니다.

## 11. 명령어 레퍼런스 (메뉴 ↔ 실제 Railway 명령)

키트가 내부에서 실행하는 실제 명령입니다. 직접 외울 필요는 없지만, 참고용으로 정리했습니다.

| 작업 | Railway 명령 | 메뉴 위치 |
|---|---|---|
| 로그인(브라우저) | `railway login` | 홈[2], RUN 1 |
| 로그인(브라우저 없이) | `railway login --browserless` | RUN 2 |
| 로그아웃 | `railway logout` | RUN 3 |
| 내 계정 확인 | `railway whoami` | 홈[3], RUN 4 |
| 버전 보기 | `railway --version` | RUN 5 |
| 업데이트 | `railway upgrade` (실패 시 `npm i -g @railway/cli`) | 홈[1], RUN 6 |
| 새 프로젝트 | `railway init` | [5]→1, RUN 7 |
| 프로젝트 연결 | `railway link` | [5]→2, RUN 8 |
| 연결 해제 | `railway unlink` | RUN 9 |
| 프로젝트 목록 | `railway list` | 홈[4], [5]→(목록), RUN 10 |
| 프로젝트 상태 | `railway status` | [5]→6, RUN 11 |
| 브라우저에서 열기 | `railway open` | RUN 12 |
| **배포(올리기)** | `railway up` | [5]→3, RUN 13 |
| 다시 배포 | `railway redeploy` | [5]→4, RUN 14 |
| 서비스 내리기 | `railway down` | RUN 15 |
| 서비스 선택 | `railway service` | RUN 16 |
| DB/플러그인 추가 | `railway add` | RUN 17 |
| 실시간 로그 | `railway logs` | [5]→5, RUN 18 |
| 빌드 로그 | `railway logs --build` | RUN 19 |
| 쉘 열기 | `railway shell` | RUN 20 |
| 명령 실행 | `railway run <명령>` | RUN 21 |
| SSH 접속 | `railway ssh` | RUN 22 |
| 플러그인 접속 | `railway connect` | RUN 23 |
| 환경변수 | `railway variables` | [5]→7, RUN 24 |
| 환경 선택 | `railway environment` | RUN 25 |
| 커스텀 도메인 | `railway domain` | RUN 26 |
| 볼륨 관리 | `railway volume` | RUN 27 |
| 공식 문서 | `railway docs` | RUN 28 |
| 도움말 | `railway help` | RUN 29 |
| 직접 명령 | `railway <내가 입력>` | RUN 30 |

> 설치는 `npm install -g @railway/cli` 를 기본으로 쓰고, Node.js가 없으면 **GitHub 릴리스의 프로그램 파일**을 자동으로 내려받아 설치합니다.

| 이런 화면/증상 | 원인 | 이렇게 하세요 |
